## タスク：Geoscope バックエンド実装 上記のプロジェクト概要を踏まえて、以下のバックエンドを実装してください。 【実装するもの】 GEOスキャンAPIエンドポイント 【仕様】 POST /api/scan リクエスト： { "brand": "自社ブランド名", "competitors": ["競合A", "競合B"], "apiKeys": { "openai": "sk-...", "perplexity": "pplx-...", "gemini": "AIza..." } } レスポンス： { "brand": "自社ブランド名", "scanDate": "2025-03-18T14:32:00Z", "results": { "chatgpt": { "score": 82, "mentions": 5, "totalQueries": 6 }, "perplexity": { "score": 74, "mentions": 4, "totalQueries": 6, "citedUrls": ["https://..."] }, "gemini": { "score": 51, "mentions": 3, "totalQueries": 6 }, "claude": { "score": 38, "mentions": 2, "totalQueries": 6 } }, "competitors": { "競合A": { "chatgpt": 88, "perplexity": 61, "gemini": 79, "claude": 70 } }, "overall": 68, "sentiment": { "positive": 81, "negative": 19 }, "rank": 2 } 【スキャンロジック】 各AIに対して以下の6種類のプロンプトを送信し、ブランド名が含まれるか確認する： 1. "【ブランド名】について教えてください" 2. "【業界】でおすすめの会社は？" 3. "【ブランド名】の評判はどうですか？" 4. "【ブランド名】と競合を比較してください" 5. "【業界】で信頼できるサービスは？" 6. "【ブランド名】はどんな会社ですか？" スコア = (ブランド名が含まれた回答数 / 総クエリ数) × 100 【技術スタック】 - Node.js（Express）またはPython（FastAPI）どちらでも可 - Vercelにデプロイできる構成 - エラーハンドリング・レートリミット対応 - 環境変数でAPIキーを管理する（ユーザーのキーはリクエスト毎に受け取る） 【注意事項】 - APIキーはサーバーに保存しない（ステートレス） - 各APIの呼び出しは並列で実行（Promise.all） - タイムアウト：30秒 - Perplexityのレスポンスからcitationsを抽出してcitedUrlsに格納 コードを実装してください。
